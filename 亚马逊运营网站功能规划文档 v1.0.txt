# 💼 我的亚马逊运营网站功能规划文档 v1.0

本网站面向跨境电商卖家，聚焦亚马逊运营效率工具、知识分享、自动化助手和个人品牌建设。前期以免费工具和资料吸引流量，后期逐步引入会员与付费机制。

---

## 🧩 顶部导航模块（建议8大区块）

### 1. 🏠 首页（Home）
- 展示我个人介绍、技能简历、项目经历
- 可作为个人品牌名片使用
- 三语切换支持：🇨🇳 中文 | 🇺🇸 英文 | 🇯🇵 日文

---

### 2. 📊 亚马逊数据分析区（Amazon Dashboard）
- 集成以下常用运营脚本（由你开发）：
  - 关键词自然排名查询工具
  - 高 ACOS 筛选工具
  - 广告数据优化器
- ✅ **页面可自由浏览介绍说明**
- 🔐 **当点击“运行脚本/生成数据”按钮时触发登录**

---

### 3. 🛠 工具集合区（Tools）
- 将实用 Python 工具前端化展示（Streamlit / Gradio 形式嵌入）：
  - 关键词生成器
  - 图片压缩器 / 尺寸工具
  - 数据清洗转换器
- ✅ **工具介绍页所有人可见**
- 🔐 **点击实际“运行工具”才触发登录**
- 后期计划：部分工具功能按点击次数免费（如每人免费用3次）

---

### 4. 🤖 AI Agent 接入区（AI Assistant）
- 整合 ChatGPT API、LangChain、mem0AI 等多种 Agent 框架
- 搭配 RAG 向量数据库（Chroma / Pinecone）进行知识检索
- 知识库内容包括：亚马逊广告、运营技巧、模板知识等
- ✅ **用户可免费体验 3 次问答**
- 🔐 **之后提示登录并引导注册/付费**

---

### 5. 📚 资料展示页（Resources）
- 分享以下内容：
  - 我的运营经验总结
  - 自制图标、模板、表格下载（如FBA费用表、广告计划表等）
  - 跨境电商常用网站导航（工具类、数据类、素材类等）
- ✅ **所有页面和内容简介可浏览**
- 🔐 **点击“下载资源”时才触发登录**

---

### 6. ✍️ 博客区（Blog）
- 写作 SEO 优化的内容吸引自然流量
  - 亚马逊广告技巧
  - 自动化工具教学
  - 精细化选品、利润计算
- ✅ 所有人可访问，无需登录

---

### 7. 🙋 用户系统（Auth）
- 注册/登录系统（支持 Email + 密码）
- 功能限制逻辑：
  - **不限制访问首页、博客、模块介绍页**
  - **当使用工具、AI、脚本、下载资源等操作时再触发登录**
- 后期升级方向：
  - 收藏功能
  - 历史记录
  - 权限区分（如普通用户 / 会员 / VIP）
  - SaaS 管理后台

---

### 8. 🌐 多语言支持（i18n）
- 支持 3 种语言切换
  - 🇨🇳 简体中文
  - 🇺🇸 英语
  - 🇯🇵 日语
- 所有文案与组件应按国际化标准组织（i18next / next-intl）

---

### 💳 预留：支付系统（未来启用）
- 💡 后期计划在点击核心功能按钮后引导收费：
  - 使用次数到达上限 → 弹窗提示订阅
  - 可集成 Stripe / Paddle / PayPal 等第三方支付
- ✅ 目前仅 **在代码和页面中预留入口**
- ❌ **暂不显示收费或强制限制使用**

---

## 📦 未来任务拆解（4大阶段）

---

### ✅ 第一阶段：前端框架搭建（Starter）

| 任务 | 描述 |
|------|------|
| ⚙️ 初始化开发模板 | 使用 Next.js App Router 架构，配置 Tailwind CSS + Shadcn UI |
| 🎨 动效风格仿制 | UI 效果参考 https://www.rimbodesigns.com/post/top-50-best-websites-2025，保留炫酷动画、加载动效 |
| 🌐 多语言配置 | 设置好 i18n 多语言切换机制（默认加载中文） |
| 🧱 页面路由结构 | 首页 / 分析区 / 工具区 / Agent区 / 资料区 / Blog / 登录页面 |

---

### ✅ 第二阶段：功能模块植入

| 功能区 | 任务 |
|--------|------|
| 数据分析区 | 集成关键词排名脚本、广告 ACOS 分析脚本 |
| 工具集合区 | 嵌入 Streamlit / Gradio 项目，支持传参运行 |
| Agent 区 | 接入 ChatGPT API + RAG 向量知识库（如 Chroma） |
| 资料页 | 上传文件到公开区 + 添加下载按钮触发登录 |

---

### ✅ 第三阶段：用户系统与权限控制

| 子任务 | 描述 |
|--------|------|
| 登录注册页面 | 邮箱 + 密码注册方式，未来可拓展 OAuth 登录 |
| 页面访问权限 | 设置访问动作时触发判断，如点击“使用工具”/“生成数据”时判断是否已登录 |
| 计次免费机制 | 可通过 cookie 或 sessionStorage 实现 AI/工具试用次数限制 |

---

### ✅ 第四阶段：支付系统预留（可选任务）

| 目标 | 描述 |
|------|------|
| 支付组件预留 | 页面中预留订阅按钮或充值提示位 |
| 支付接口接入（后期） | Stripe / PayPal 等付费渠道配置，设置不同权限级别（Free / Pro） |
| 后台管理 | 管理用户权限与付费状态（后续可对接 Supabase / Clerk / Firebase 等）

---
